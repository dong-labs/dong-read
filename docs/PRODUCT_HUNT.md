# 读咚咚 (Read) - Product Hunt 发布文案

## 标题
Read - 本地、私有的个人知识数据层，为 Agent 时代设计

## 描述

读咚咚是一个个人知识数据层，支持多种客户端访问（CLI、Python SDK、MCP Server）。

当你看到一句有启发的话、一篇好文章，快速存下来。CLI、浏览器插件、Agent 都可以访问这些数据。

## 核心特点

1. **本地私有** - 数据存放在 `~/.read/read.db`，不上云、不同步、不追踪
2. **Agent 友好** - JSON 输出 + Python SDK，Agent 可以直接 `import`
3. **极简核心** - 只做收集，不做整理。标签、搜索、AI 摘要都由客户端实现
4. **数据层优先** - Core Library 是核心，CLI 只是其中一个客户端

## 使用场景

- 看到一句好话 → `dr add "这句话说得真好"`
- 收藏一篇好文章 → `dr add --url "https://example.com"`
- 让 Agent 推荐历史摘录 → `client.search("AI")`

## 与其他工具的区别

| 工具 | 定位 | 数据位置 | Agent 访问 |
|------|------|----------|-----------|
| Cubox | 收集工具 | 云端 | ❌ |
| Notion | 笔记工具 | 云端 | ✅ API |
| **Read** | **数据基础设施** | **本地** | **✅ SDK + MCP** |

## 讨论

**问题 1：为什么不做标签？**
答：先验证核心需求，够用就行。标签可以由客户端实现。

**问题 2：为什么是 CLI？**
答：CLI 是"核心接口"，不是唯一入口。未来会有浏览器插件、MCP Server 等。

**问题 3：数据安全吗？**
答：完全本地存储，你可以随时复制 `~/.read/read.db` 进行备份。
