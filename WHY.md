## Agent 时代正在到来

你可能注意到了：

- **Claude Code** - Anthropic 官方的 CLI 编程助手
- **OpenInterpreter** - 本地运行代码的 AI
- **Cursor** - AI 原生编辑器
- **warp**️ - AI 原生终端

**Agent 越来越多，但基础设施跟不上。**

这些 Agent 很强大，但它们需要**可操作的数据**。

你的摘录、你的收藏、你的想法——如果都在云端 SaaS 里，Agent 怎么访问？

如果数据在本地的 SQLite 里，一行命令就拿到了：

```bash
read ls --limit 5
# → 返回 JSON
# → Agent 解析、分析、推荐
```

**这就是 Agent 原生。**

## 读咚咚是什么

读咚咚是**个人知识数据层的 Python 库**，提供 CLI、SDK 和 MCP Server 等多种访问方式。

```
┌─────────────────────────────────────────┐
│  你看到一句好话 / 一篇好文章              │
├─────────────────────────────────────────┤
│  read add "这句话说得真好"                │
│  或浏览器插件一键收藏                     │
├─────────────────────────────────────────┤
│  存入 ~/.read/read.db (SQLite)            │
├─────────────────────────────────────────┤
│  Agent 随时可以读取、分析、推荐           │
└─────────────────────────────────────────┘
```

**核心特点：**

1. **数据层优先** - Core Library 是核心，CLI/插件/SDK 都是客户端
2. **本地私有** - 数据在你电脑上
3. **Agent 友好** - JSON 输出 + MCP Server + Python SDK
4. **极简核心** - 数据层只做存储和接口

## 谁需要读咚咚

### 1. Agent 用户

你用 Claude Code、Cursor、ChatGPT 等 AI 工具。

你需要一个 Agent 能"看见"的知识库。

读咚咚的数据，Agent 可以直接读取：

```python
# 方式1: Python SDK（推荐）
from read import Client
client = Client()
items = client.search("AI").limit(5)

# 方式2: MCP Server（Claude/ChatGPT 原生支持）
# 在 Agent 中直接调用 read_search、read_add 等工具

# 方式3: CLI（兼容性）
import subprocess
result = subprocess.run(["read", "ls", "--limit", "10"], capture_output=True)
data = json.loads(result.stdout)
```

### 2. 程序员/终端用户

你大部分时间在终端里。

切换到浏览器、打开 Notion、点来点去——太慢了。

一行命令搞定，是程序员的方式。

### 3. 知识工作者

你每天看很多文章、书、视频。

你不想让这些好内容流失，但也不想被复杂的笔记软件绑架。

你需要一个"丢进去就行"的地方。

```bash
# 这句话来自某篇文章
read add "Agent First, Human Second" \
    --url "https://example.com" \
    --source "CLAUDE.md"
```

### 场景 4：让 Agent 推荐

```bash
# 早上，让 Agent 推荐一条昨天的摘录
read ls --limit 1 --type content
# → Agent 拿到结果，生成早安问候
```

## 设计哲学

### 1. 收集优先，整理稍后

大多数人死于"整理"，而非"收集"。

先让它流动起来，分类是以后的事。

### 2. Agent First, Human Second

人类可以直接用，但更要让 Agent 好用。

JSON 输出、简单的命令、清晰的接口。

### 3. 本地私有

你的想法、你看到的、你收藏的——这些都是你私人的。

为什么要上传到别人的服务器？

### 4. 极简主义

> "有疑问时，不加。想加新功能？先划掉一个旧的。"

不做标签、不做自动抓取、不做全文索引、不做 AI 摘要。

只做一件事：**让你快速把内容存进来。**
